Why this sequence matters: You cannot build AI agents without clean data. You cannot get clean data without building the pipelines first. The marketing data layer comes first because (1) it is the most business-critical right now and (2) it is the simpler half — external APIs with clear schemas. Ops data (FieldBoss, Dynamics) is messier and requires more engineering. Do marketing right, then carry the same patterns to ops.

This is the clean data structure you build in May. Every column is intentional. Every table serves either Power BI dashboards today or AI agent training data tomorrow. Build it right once.

The most important join in the entire data model:
campaign_daily_metrics.campaign_id = lead_attribution.campaign_id = call_tracking.campaign_id

This three-way join is what produces the report leadership wants: "We spent $4,140 on Hotel FL campaign → drove 31 calls (18 answered) + 23 form fills → 4 became signed contracts → $87,000 ARR → ROAS 21x." Without clean campaign_id preserved all the way through the UTM chain, this report is impossible to build.

MongoDB + Voyage AI — how they fit in this stack

MongoDB AtlasStores structured metadata alongside vector embeddings — lead profiles, campaign history, client interaction logs. Works alongside Qdrant, not instead of it.
Voyage AIGenerates text embeddings for marketing content, review text, client communications. Better performance than OpenAI embeddings for retrieval tasks. Used for: review similarity search, lead context retrieval, campaign content RAG.
Qdrant (on Railway → Azure)Vector store for agent memory. Voyage AI creates the vectors, Qdrant stores and retrieves them. Agents retrieve past context during reasoning cycles.
VercelHosts any client-facing web interfaces (lead capture forms, reporting portals, dashboard embeds) connected to Fabric data via API routes.
Railway (dev phase)Hosts LangChain/LangGraph agents during development and testing. Cheaper and faster to iterate than Azure Container Apps. Migrate to Azure once agents are production-stable.
