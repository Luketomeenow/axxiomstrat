Predictive Maintenance AI system for Axxiom Elevator — data sources, risk scoring, alerts, and workflow integration

Step 1 of 5

Where does the prediction data come from?

The model is only as good as its inputs. Six data sources feed the risk score — all already available in Axxiom's Dynamics environment or IoT layer.

Service history log

Every past work order per asset — repair type, fault codes, parts replaced, time-to-fix. Stored in Dataverse already.

PM interval tracking

Days since last preventive maintenance vs. manufacturer-recommended interval. Drift beyond interval = rising risk.

Parts & component age

Installation dates and expected lifespans for motors, cables, door operators, buffers. Flags components near end-of-life.

IoT sensor data

Optional: vibration, door cycle count, motor temperature, current draw. Feeds real-time anomaly detection on top of the baseline model.

Usage intensity

Trips-per-day by building type — a hospital elevator runs 10× harder than a residential one. Model weights risk by actual load, not just age.

Environment & age

Install year, coastal/humid environment exposure, building type. Older units in coastal FL degrade faster than newer inland units.

Phase 1 runs on Dynamics data only — no IoT needed. IoT sensors are added in Phase 2 for clients willing to pay for real-time monitoring as a premium tier.

How the risk score is calculated

Azure Machine Learning runs a weighted scoring model against each elevator asset. The output is a 0–100 failure risk score, recalculated every 24 hours.

How it plugs into Dynamics 365 Field Service

The predictive layer sits on top of the existing Dynamics infrastructure — it generates work orders and updates asset records automatically, with no new interface for the ops team to learn.

Daily automated workflow — Power Automate

Score recalculates (6am daily) → Critical assets flagged → Draft WO created → Ops manager notified
Manager approves WO → Enters dispatch queue → Compliance gate → Tech assigned

Step 5 of 5

How this system generates revenue, not just efficiency

Predictive maintenance isn't just an ops tool — it's a client-facing product and a natural trigger for Axxiom's highest-value service lines.

Modernisation pipeline

Any asset scoring above 70 for 90+ consecutive days gets flagged as a modernisation candidate. The account manager receives a templated recommendation to present to the client — moving from a repair conversation to a capital project conversation.

Premium monitoring tier

IoT sensor data (real-time vibration, temperature, door cycles) is packaged as an add-on to maintenance contracts. Clients pay a monthly fee for a live risk dashboard and real-time alerts — recurring revenue on top of the service contract.

Quarterly asset health report

AI-generated report per client showing risk scores, trend over time, completed PMs, and upcoming recommendations. Sent automatically each quarter — positions Axxiom as a proactive partner, not just a reactive vendor. Strong retention tool.

Emergency reduction as a KPI

Track emergency call volume per client before and after predictive maintenance is active. Present the reduction as a dollar-value savings — every emergency avoided is roughly $2,000–$5,000 in avoided repair costs plus downtime losses. This is the renewal conversation.
