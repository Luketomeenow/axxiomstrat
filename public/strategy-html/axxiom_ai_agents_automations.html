Axxiom Elevator AI agents and automations — GHL, Dynamics, FieldBoss, Google Analytics to Power BI

System architecture

Microsoft Dynamics = single source of truth. Everything flows in.

All four systems feed data into Dynamics Dataverse. Power Automate is the orchestration layer. Power BI reads from Dynamics for the data analyst. GHL handles client-facing marketing execution. FieldBoss is the field service engine. AI agents sit between the systems and make decisions.

Why Dynamics must be the master: Compliance requires every dispatch decision, tech certification check, client communication, and financial event to have an audit trail. Dynamics Dataverse provides that — every record has a created-by, modified-by, and timestamp automatically. Power BI then reads this clean, governed data directly without any intermediate transformation that could break the audit chain.

Marketing AI agents — 7 automations

Turn ad spend and lead data into Dynamics-governed intelligence

Every marketing event — a lead form fill, a Google Ads click, a GHL conversation — must land in Dynamics with full attribution. These agents make that happen automatically and add AI intelligence on top.

Operations AI agents — 6 automations

FieldBoss + Dynamics + AI = an ops system that runs itself

Operations data is the most valuable data Axxiom has — and right now it's locked inside FieldBoss. These agents extract it, govern it in Dynamics, and give the ops team decision-making intelligence they don't currently have.

Ops Agent 1 — FieldBoss Work Order Sync (FieldBoss → Dynamics)

Every work order, tech, asset, and job outcome lands in Dynamics immediately — the data foundation for every other ops automation

Build first Operations
TRIGGER: FieldBoss webhook fires on: work order created · work order updated · job status changed · tech checked in/out · job closed with notes Real-time
Make maps FieldBoss WO fields → Dynamics "ax_WorkOrder" entity. Key fields: WO number, asset ID, client ID, tech assigned, service type, start/end time, outcome, fault codes, parts used, tech notes Map fields
AI fault code extractor: reads tech notes (free text) → extracts structured fault codes using ASME A17.1 taxonomy → writes fault_codes[] and components[] fields to Dynamics asset service record. Confidence score attached — low confidence → human review queue AI extract
Asset record in Dynamics updated: last_service_date, last_fault_code, service_count, days_since_last_PM recalculated. Risk score recalculated by predictive model. Compliance timeline checked. Asset update
Power BI: ops analyst sees live job completion rates, first-fix rate, avg job duration, fault code frequency, tech performance by market. All governed, timestamped, audit-ready in Dynamics. Power BI
FieldBoss webhooks / API Make Azure OpenAI (fault extraction) Dataverse API Power Automate (asset update)

Ops Agent 2 — Predictive Maintenance AI Agent

Score routing: 75-100 (Critical) → draft PM work order auto-created in FieldBoss + Dynamics + Teams alert to ops manager + account manager notified to contact client. 50-74 (Watch) → flagged on Dynamics risk dashboard. Below 50 → log only. Route
AI-generated job brief for tech: asset history summary, top 3 most likely fault components, suggested parts to bring (from past similar jobs), last 3 fault codes, safety notes for site type. Written to FieldBoss work order notes field. AI job brief
Power BI: fleet risk heatmap by market, risk score trend over time, # of emergency call-outs prevented (tracked by comparing flagged units vs actual emergency rate), modernization pipeline candidates. Power BI
