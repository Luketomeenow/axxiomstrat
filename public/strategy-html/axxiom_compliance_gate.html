Compliance gate checks in the Axxiom Elevator dispatch agent — four sequential checks before technician assignment

Work order created → Compliance gate → Smart dispatch engine → Tech assigned

Four sequential checks — all must pass

1 Elevator mechanic license — active & valid Hard block

License number on file in Dataverse

Each technician profile in Dynamics stores their state elevator mechanic license number, issue date, and expiry date. This is mandatory to onboard any tech.

Expiry check against today's date

The gate compares expiry date against the current date. If expired — or expiring within 14 days — the tech is flagged and excluded from dispatch candidates.

License valid in job's state

Some states require a state-specific license in addition to a general mechanic cert. The check maps the job site's state against the tech's licensed states list — not just "has a license" but "licensed here."

If license expired or not valid for this state — tech is hard-blocked. No override. Escalates to compliance officer.
2 AHJ jurisdiction match Hard block

AHJ lookup from job site address

The work order's site address is resolved to its Authority Having Jurisdiction — could be a state board, county, or city depending on location. This is pulled from a curated AHJ reference table in Dataverse, not assumed from the state alone.

Tech's AHJ approvals checked

Each tech profile stores which AHJs they are registered or approved under. Miami-Dade County, NYC Dept of Buildings, Chicago DOB — all different. Tech must be listed as approved for the specific AHJ of the job site.

AHJ registration renewal tracking

AHJ registrations expire independently of mechanic licenses. The gate flags any registration expiring within 30 days and alerts the compliance team — but does not block dispatch unless already expired.

Tech not registered under job's AHJ — hard block. Next eligible tech in queue is evaluated instead.
3 Site-specific clearance Soft block

Healthcare — background check required

Hospitals and healthcare facilities often require a criminal background check on file. The tech profile stores background check status and date. If the site is tagged as healthcare and the tech has no cleared background check, the gate flags this for dispatcher review.

Transportation hubs — SIDA / security badge

Airport and transit sites require a SIDA or security access badge. The tech profile stores badge numbers and expiry. Gate checks badge validity against site type tag on the work order.

Government / federal sites — clearance level

Some government buildings require security clearance levels. The gate checks the site's required clearance tier against the tech's stored clearance level before allowing dispatch.

Soft block — dispatcher is alerted and must manually approve or reassign. Does not auto-reject.
4 Insurance & indemnity coverage Advisory

General liability coverage active

The gate checks that Axxiom's general liability policy covers the job site's state. Relevant when entering a new market where coverage may not yet be extended.

Client contract on file

Checks that the client has a signed service agreement in Dynamics before dispatching. Prevents sending a tech to an uncontracted site — a billing and liability risk.

Advisory only — flags appear on the work order for dispatcher awareness but do not block dispatch.

Override policy

Gates 1 & 2 — zero override

License expiry and AHJ mismatch are regulatory requirements. No dispatcher, manager, or system condition can override these blocks. If no eligible tech exists, the system escalates to the compliance officer to source a qualified sub-contractor.

Gate 3 — manager override only

Site-specific clearance blocks require a named manager to approve in Dynamics before the work order proceeds. The approval is logged with timestamp and reason — creating a full audit trail.

Gate 4 — dispatcher discretion

Insurance and contract flags are informational. Dispatchers see them and can proceed. The flags are logged on the work order for billing and legal reference.
