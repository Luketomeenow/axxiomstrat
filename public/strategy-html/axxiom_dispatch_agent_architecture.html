Dispatch and Work Order Agent architecture for Axxiom Elevator on Microsoft Dynamics 365

Axxiom Elevator — Dispatch Agent

AI Dispatch & Work Order Agent

Built natively on Microsoft Dynamics 365 Field Service — the compliance-safe approach that augments what they already have rather than replacing it.

Compliance considerations locked in by design

  • All data stays inside Microsoft's ecosystem — no external AI SaaS touching work order or client data
  • Technician certifications and AHJ (Authority Having Jurisdiction) requirements enforced before dispatch
  • Full audit trail on every dispatch decision — who was assigned, when, and why
  • Role-based access control via Azure AD — AI agent acts with scoped permissions only

Core tech stack

Dynamics 365 Field Service Copilot Studio Power Automate Azure OpenAI Dataverse Azure Maps Teams / SMS

System architecture — 4 layers

1 Service Request Intake & Classification Entry point

Multi-channel intake

Phone, email, web form, or client portal — all routes into a single Dynamics work order queue

AI job classifier

Azure OpenAI reads the request and tags it: Repair / Maintenance / Inspection / Emergency — with urgency score

Work order auto-creation

Dynamics Field Service work order created with pre-filled service type, asset ID, site location, and SLA tier

2 Compliance Gate — Pre-Dispatch Checks Critical layer

Cert & license check

Validates tech's elevator mechanic license is active and valid for the job's AHJ jurisdiction — hard block if expired

Jurisdiction matching

Matches job location to correct code authority — prevents sending a tech not certified under that state/county AHJ

Safety clearance

Checks site safety requirements (healthcare, transportation hubs) against tech's background/clearance status

3 Smart Dispatch Engine Core AI logic

Geo-optimization

Azure Maps ranks available techs by proximity + current route — minimizes drive time and overtime risk

Capacity balancing

Checks current workload, open jobs, and scheduled hours per tech — avoids overloading high performers

Asset familiarity score

Prefers techs who've previously serviced that specific elevator unit — reduces diagnosis time on complex repairs

Human override

Dispatcher sees AI recommendation with reasoning — one click to approve or reassign. AI suggests, human approves.

4 Job Brief Delivery & Closeout Output layer

Tech job brief — SMS/Teams

AI-written job brief pushed to tech: site address, elevator model, last service notes, parts to bring, safety notes

Client status updates

Auto-sends ETA notification to client when tech is dispatched, and follow-up when job is marked complete

AI closeout summary

Tech submits field notes → AI generates service report, updates asset history in Dynamics, flags any follow-up needed
